man - мануал;

--help - список возвожностей;

apt - пакетный менеждер;

cat - прочитать файл;

head - прочитать начало файла;

tail - прочитать конец файла;

echo - принимает на вход строчку и выдаёт её в STDOUT, который уже можно перенаправлять;

wc (word count) - считает колличество слов;

uniq - убирает дублирующие слова;

sort - входные данные сортируются и выводятся на экран;

mv (move) - В *nix-системах не существует понятия "переименовать файл". Переименование всегда равносильно перемещению;

cp (copy) - копирование файлов;

mkdir (make directory) - создание директории;

clear - очистить экран терминала;

touch - Основная задача утилиты поменять время последнего доступа к файлу, но она обладает побочным
эффектом. Если файла не существует, то он будет создан — именно поэтому её используют
для создания файлов, хотя это не основное предназначение.;

tree - Команда tree не зря получила такое название — она умеет изображать структуру всех каталогов,
имеющихся на компьютере, в виде дерева. (установка "sudo apt-get install tree");

env (environment) - выдает список переменных окружения в вашей локальной;

history - вывести историю bash комманд;

whoami - позволяет выяснить, кто же я такой;

ps (process status) - выводит отчёт о работающих процессах
Информацию о том, какой процесс и под каким пользователем запущен, можно получить из вывода "ps aux";

sudo (substitute user and do) — дословно "подменить пользователя и выполнить";
sudo -i - вход для root;

id - Посмотреть свой идентификатор;

stat - статистика папки;

type - узнать, где лежит исполняемый файл конкретной программы;

whereis - узнать, где лежит исполняемый файл конкретной программы;

which - узнать, где лежит исполняемый файл конкретной программы;
